**Goal:** Create `.claude/skills/review-pr/SKILL.md` — a reusable Claude Code skill that guides consistent PR reviews for this repository.

**Architecture:** A single markdown file read by the Claude Code `Skill` tool. No code generation, no tests. The skill is a pure checklist covering the four concerns that matter for this project: API↔web sync, generated files, domain correctness, and code quality.

### Task 1: Create the skill file

- Create: `.claude/skills/review-pr/SKILL.md`

- [ ] **Step 1: Create the directory and skill file**

Create `.claude/skills/review-pr/SKILL.md` with exactly this content:

```markdown
# PR Review

Fetch the PR, then work through the checklist. Report findings grouped by section. End with a verdict.

## 1. Fetch

```bash
gh pr view <number> --comments
gh pr diff <number>
```

## 2. API ↔ Web sync

- [ ] `api/src/app.ts` or a Zod schema changed → `api/openapi.json` must be updated
- [ ] `api/openapi.json` changed → `web/src/api/generated/hooks.ts` must be regenerated by Orval (`npm run generate`)
- [ ] `web/src/api/generated/**` must not contain hand-edits (Orval has `clean: true` — any manual change will be wiped)

## 3. Generated files

- [ ] `api/prisma/schema.prisma` changed → Prisma client must be regenerated
- [ ] `npm run generate` was run after any schema or route change
- [ ] No generated file has manual edits layered on top of generated output

## 4. Domain correctness

- [ ] Zod schemas match the resource/booking domain: sensible field names, types, required/optional split
- [ ] Validation is enforced at the API boundary (route input schemas), not only in the frontend
- [ ] No validation gaps: missing-record cases, bad enum values, empty strings treated as valid

## 5. Code quality

- [ ] Change is small and focused — one concern per PR
- [ ] No obvious bugs
- [ ] Edge cases handled: resource not found, invalid input
- [ ] No risky assumptions that could silently break in production

## 6. Verdict

Approve, request changes, or flag blockers. Reference specific `file:line` for each issue.
```
